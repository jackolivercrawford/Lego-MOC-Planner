## Design Context

### Users
The primary user is one person sorting and checking physical LEGO lots at a desk. They need to move quickly between cabinet locations, part/color lots, and inventory states while keeping the physical workflow clear and low-friction.

### Brand Personality
Calm, utilitarian, precise. The interface should feel dependable and organized rather than playful for its own sake.

### Aesthetic Direction
Use a light-mode, dense-but-readable operational UI with restrained panels, stable grids, compact controls, and status color used only where it improves scanning. Avoid marketing-style layout, decorative visuals, and color-heavy surfaces that compete with the actual LEGO inventory state.

### Design Principles
- Make inventory state visible at every useful scale, from global overview to drawer to individual lot.
- Preserve physical sorting speed with compact controls, stable layout, and direct status changes.
- Pair color with text, labels, or accessible names so status is not color-only.
- Keep responsive layouts stable and readable on narrow screens without overlapping controls.
- Persist workflow preferences locally when they affect repeated checking work.
